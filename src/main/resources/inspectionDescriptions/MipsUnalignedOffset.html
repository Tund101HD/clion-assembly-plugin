Reports memory accesses whose constant offset is not aligned to the access size.

Word and half-word memory instructions require naturally-aligned addresses on MIPS: a 4-byte access (lw, sw, ll, sc) needs an offset divisible by 4, and a 2-byte access (lh, lhu, sh) needs an even offset. A misaligned constant offset triggers an address-error exception at run time.

For example: 


    lw      $t0, 3($sp)     ; word access needs a 4-aligned offset
    lh      $t0, 1($sp)     ; half-word access needs an even offset

Only constant offsets are checked; register-relative alignment cannot be known statically. Byte accesses (lb, sb) have no alignment requirement and are never flagged.
