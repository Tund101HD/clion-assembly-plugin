Reports operand-size mismatches between the operands of a single instruction.

Most x86 instructions require their register and memory operands to share the same width. This inspection flags combinations like an 8-bit register with a 64-bit register, or a sized memory operand that disagrees with its register counterpart — which the assembler rejects as a size mismatch.

For example: 


    mov     al, rbx     ; 8-bit destination, 64-bit source
    add     eax, rbx    ; 32-bit and 64-bit operands

Sign/zero-extending moves (movsx, movzx, movsxd) deliberately mix sizes and are not flagged.
